# Food Recipe App

This is a Flutter + Firebase recipe app. Users can register, log in, browse recipes by cuisine, open recipe details, and post new recipes to Firestore.

## How The App Works

The app starts in `main.dart`, initializes Firebase, loads the shared theme, and opens the splash screen.

From there the user flow is:

1. `SplashScreen` shows the landing UI.
2. User goes to `LoginScreen` or `RegisterScreen`.
3. After authentication, the app opens `HomeScreen`.
4. `HomeScreen` loads recipe data from Firestore through `RecipeRepository`.
5. Tapping a recipe opens `RecipeDetailScreen`.
6. The center footer action button opens the small action menu:
   - middle button: post recipe
   - right button: saved recipes
   - left button: placeholder

## Data Model

Each recipe document in Firestore is expected to look like this:

```json
{
  "id": "VN-01",
  "title": "Pho Bo",
  "category": "Vietnamese Food",
  "imageUrl": "https://...",
  "rating": 4.7,
  "description": "....",
  "durationMinutes": 30,
  "calories": 420,
  "difficulty": "Easy",
  "origin": "Vietnam",
  "ingredients": ["..."],
  "steps": ["..."],
  "isPopular": true,
  "createdAt": "server timestamp"
}
```

The home page now uses the selected cuisine to drive three rows:

- `Category` row: all recipes in the selected cuisine, shown in random order
- `Featured` row: newest 5 recipes in the selected cuisine
- `Popular Recipes` row: only popular recipes in the selected cuisine, shown in random order

## Main Code Files

### App Setup

- `lib/main.dart`
  Bootstraps Flutter, initializes Firebase, enables `DevicePreview` in non-release mode, and starts `RecipeApp`.

- `lib/firebase_options.dart`
  Generated by FlutterFire. Holds the Firebase config for Android, iOS, macOS, web, and Windows.

- `lib/core/app_theme.dart`
  Defines shared app colors, asset paths, and the base theme used across screens.

### Model

- `lib/models/recipe.dart`
  The main recipe model. Includes:
  - constructor
  - `fromMap`
  - `toMap`
  - `copyWith`

### Data And Services

- `lib/data/recipe_repository.dart`
  Firestore data access layer for recipes. Handles:
  - fetching recipes by category
  - fetching newest recipes by category
  - fetching popular recipes by category
  - fetching a recipe by id
  - creating a new recipe

- `lib/services/auth_service.dart`
  Firebase authentication service. Handles:
  - email/password register
  - email/password login
  - Google sign-in
  - sign out
  - user profile upsert into Firestore

### Screens

- `lib/screens/splash_screen.dart`
  First screen users see. Uses the splash image background and routes users to login or register.

- `lib/screens/login_screen.dart`
  Email/password login screen with Google sign-in option.

- `lib/screens/register_screen.dart`
  Registration form with password confirmation and Google sign-in option.

- `lib/screens/home.dart`
  Main browsing screen. Responsibilities:
  - shows greeting header
  - cuisine tabs
  - recipe rows for category, featured, and popular
  - footer action menu with blur overlay
  - logout action

- `lib/screens/recipe_detail_screen.dart`
  Loads a recipe by id and shows:
  - hero image
  - title and rating
  - category and description
  - stats
  - ingredients
  - cooking steps

- `lib/screens/post_recipe_screen.dart`
  Recipe creation form. Lets the user enter all fields required for a recipe and save it to Firestore.

- `lib/screens/saved_recipes_screen.dart`
  Placeholder screen for saved recipes.

### Reusable Widgets

- `lib/widgets/common_widgets.dart`
  Shared UI building blocks:
  - `AssetImageWithFallback`
  - `RoundedInput`
  - `PrimaryButton`
  - `SecondaryButton`
  - `GoogleButton`
  - `RecipeNetworkImage` for cached remote recipe images

- `lib/widgets/recipe_card.dart`
  Shared recipe card used in the home page rows.

- `lib/widgets/detail_stat_chip.dart`
  Small stat widget used in the recipe detail screen.

## Firestore Collections

The app currently depends on:

- `recipes`
  Main recipe collection used by home, detail, and post recipe flows.

- `users`
  User profile collection written by `AuthService`.

## Notes

- The home screen cuisine filters depend on Firestore categories being exactly:
  - `Vietnamese Food`
  - `Chinese Food`
  - `Japanese Food`
  - `Italian Food`

- `fetchNewestRecipesByCategory` tries to order by `createdAt`. If older documents do not have that field or the required Firestore index is missing, the repository falls back to a simpler fetch.

- Recipe images are loaded with `cached_network_image` to improve repeat-load speed and reduce visible flicker.
